#클라이언트 점검사항
-소켓 연결 끊기는지 점검 및 수정

1. FindIDController/FindPWDController
아이디/비밀번호 찾기 시 도중 뒤로가면 클라이언트 안끊기는 문제 수정

2. FindRoomController -> controller.setData(데이터, id)
방 데이터를 컨트롤러에 전달
-> 전달하고 바로 서버 연결해서 이미지 가져온 뒤에 소켓 닫기처리 추가

3. 예약정보 조회 시 아무것도 선택안했을 때 오류 처리

4. 결제 시 정보 유효하지 않으면 오류 메시지 출력

5. 예약 가능 시간 조회 시 가능한 시간이 없는 경우
-> 처리 수정

5. 결제 취소 버튼 눌렀을 때 서버로 요청한 뒤 소켓 연결 끊기
-> 이후 MyPage 갱신하도록 요청

# 추후 개선이 필요한 부분

0. 시간이 없어서 급하게 성공 케이스만 동작

1. 실패 케이스에 대한 대처
-> 파일 쓰기 처리 실패 시 DB 롤백 X

2. HomeCardLayout -> VBox createCard
요청할 때 방 ID가 아닌 pic를 사용하면 DB 변경 시 대처 어려움
ID로 요청해서 서버에서 알아서 파일 주소를 서버에 건네주고
서버에서 해당 주소에 있는 파일들을 가져오도록 해야 함.

FileTransferCommand -> getRoomImg
여기에서 방 ID로 요청하면 DB에서 서버로 주소를 보내고
서버는 해당 경로의 사진 정보를 클라이언트에 보내도록 해야 함.

3. 스케쥴
-> 등록된 날짜만 사용할 수 있음.
-> DB에 날짜가 등록되어있지 않으면 사용불가.
-> 스케쥴 조회를 요청했을 때 스케쥴 정보가 
없으면 새로 생성하고 그 정보를 넘겨줘야 함.
-> 오전/오후/야간 3개 정보를 DB에 담는 것은 비효율적
-> 시간 정보가 3가지라면 "111" or "000" 처럼 
문자열로 각 시간 정보를 표현해도 됨
-> 이렇게 하면 24시간일 경우 24개 문자열로 확장 가능함.
-> 시간 타입에 대한 조건은 DB가 아닌 서버측에서 정해야 함.
-> 시간이 오전을 지났으면 제외하고 나머지 정보만 보여주도록.

4. 동시성 제어
DB 트랜잭션 및 동시성 제어 수행 필요
